# ソーシャルゲームシステム設計 — 学マス参考

## 背景

学園アイドルマスター（学マス）の優れたゲームデザインを、定食屋ADV「灯」の世界観に翻訳する。
搾取的でなくプレイヤーを尊重する設計哲学を維持しつつ、戦略性・周回性・収集欲を強化する。

## 学マス → 灯 翻訳マップ

| 学マス | 灯 | 設計意図 |
|--------|-----|----------|
| Vo/Da/Vi 3軸 | 信頼/理解/共感 | 単一親密度→多軸化で戦略性を生む |
| スキルカード | 行動カード | 毎日の手札から選ぶ意味ある選択 |
| プロデュース1回 | 1季節（30日） | 季節サイクルで周回性を生む |
| 中間/最終試験 | 季節の節目イベント | パラメータ到達度でシーン分岐 |
| メモリー | 思い出のかけら | 条件達成で獲得する収集要素 |
| サポートカード | 装備メモリー | 思い出をセットしてパッシブ効果 |
| SPレッスン | 特別な日（天候/季節） | ランダム性のある高報酬機会 |
| 親愛度 | 親密度（3軸の合計） | 後方互換を保つ |

## 1. 関係性3軸パラメータ

学マスの Vocal/Dance/Visual に対応する3軸。
既存の単一 `intimacy` を分解し、戦略的な行動選択を促す。

```
信頼 (Trust)    — 丸山との会話、日々の積み重ねで育つ
理解 (Understanding) — 観察、ひよりの仕草や好みを知ることで育つ
共感 (Empathy)  — 食事を共有し、同じ体験をすることで育つ
```

**親密度との関係**: `intimacy = trust + understanding + empathy`（後方互換）

**章解放条件の拡張**:
- 既存: `unlock_intimacy: 100`（合計のみ）
- 新規: `unlock_condition: { trust: 30, understanding: 20, empathy: 50 }`
  - 特定軸の到達が必要な章もある（例: ch4は理解80必須）

## 2. 行動カードシステム

学マスのスキルカードに対応。「毎日同じ4択」から「手札から戦略的に選ぶ」へ。

### カードの構造
- 名前・説明（テーマ性のあるフレーバーテキスト）
- AP コスト（1-3）
- 3軸への影響値（信頼+X, 理解+Y, 共感+Z）
- 解放条件（初期/ランク到達/章クリア/季節限定/天候限定）
- レアリティ（通常/上質/特別 — 出現率に影響）

### 手札の仕組み
1. 毎朝、所持カードプールからランダムに **4枚** ドロー
2. 天候・季節で特定カードの出現率UP
3. AP の範囲内で手札からカードをプレイ
4. 使わなかったカードは翌日に持ち越さない（毎日新鮮な選択）

## 3. 季節・天候システム

### 季節サイクル（春→夏→秋→冬、各30日）
- `season = (day - 1) / 30 % 4` で決定
- メニュー・カード・イベントが季節に連動
- 各季節末（30日目）に **季節の節目イベント** が発生

### 天候（毎日変動）
- 晴れ(50%) / 曇り(25%) / 雨(20%) / 雪(冬のみ5%)
- 天候でカードの出現率が変化し、限定カードが手札に入る
- 雨の日は「灯」の雰囲気が変わる（テキスト差分）

### 季節の節目イベント（学マスの中間/最終試験）
- 3軸パラメータの到達度に応じて **S/A/B/C ランク** 判定
- ランクごとに異なるシーン（ご褒美ストーリー）
- Sランク条件は「3軸バランスよく高い」= 学マスの True End 条件

```
Sランク: trust >= 40 AND understanding >= 40 AND empathy >= 40
Aランク: 合計 >= 100
Bランク: 合計 >= 60
Cランク: それ以外
```

## 4. 思い出（メモリー）システム

学マスのメモリーに対応。特定条件で獲得する収集要素 + パッシブ効果。

### 獲得条件の例
| 思い出 | 条件 | パッシブ効果 |
|--------|------|-------------|
| 「最初の日替わり」 | 初めて食事をする | 共感+2/日 |
| 「丸山の笑い話」 | Talk を 5回 | 信頼+2/日 |
| 「ページをめくる指」 | Observe を ★2以上で | 理解+3/日 |
| 「雨の日の灯」 | 雨の日に来店 | 全+1/日 |
| 「隣の席の温度」 | ★4 到達 | 全+2/日 |
| 「春の節目 Sランク」 | 春節目でS | 共感+5/日 |

### 装備スロット
- **3枠**（信頼/理解/共感 各1枠）
- セットした思い出のパッシブ効果が毎日の行動に加算
- 組み合わせ戦略が生まれる（3軸のどこを伸ばすか）

## 5. 拡張デイリーミッション

既存の3種から、カードシステム・3軸に対応した形に拡張。

### 固定ミッション（毎日）
- 灯で食事をする (+10 共感)
- 丸山と話す (+5 信頼)
- 3つの行動をする (+20 分散)

### 週替わりミッション（新規）
- 「理解を30上げる」→ 報酬: 限定カード1枚
- 「3種類の行動をする」→ 報酬: 思い出のかけら
- 「季節メニューを全制覇」→ 報酬: 季節限定テキスト

## 6. ゲームループ全体像

```
【ミクロ】 1アクション（30秒〜1分）
  手札からカード選択 → AP消費 → 3軸パラメータ上昇 → テキスト表示
  ★ 「どのカードを使うか」が毎回の意味ある選択

【デイリー】 1日の灯訪問（5〜10分）
  朝: 天候確認 → 手札ドロー → 灯に行く
  昼: AP5の中でカードを選んでプレイ（3〜5回）
  夕: ミッション確認 → 次の日へ → 「明日は雨か、あのカード来るかな」
  ★ セッション終了時に明日への期待がある

【シーズン】 1季節（30日）
  日々の積み重ね → 3軸パラメータ成長 → 章解放（ストーリー報酬）
  季節末: 節目イベント → ランク判定 → ご褒美シーン
  ★ 「次の季節はAランク取りたい」という目標

【マクロ】 1年サイクル（120日）
  春→夏→秋→冬 → 2年目春…
  全思い出収集 / 全Sランク / 全章クリア / True End
  ★ コンプリート欲とやり込み
```

## 7. 既存システムとの互換性

- `intimacy` は `trust + understanding + empathy` の合計値として維持
- 既存の `ActionType` (Eat/Observe/Talk/Special) はカードの `category` として残す
- `unlock_intimacy` は合計値チェックとして引き続き動作
- デイリーミッションの構造は維持しつつ報酬を3軸対応に
