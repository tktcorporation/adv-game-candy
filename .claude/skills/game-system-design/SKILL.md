# ゲームシステム設計スキル — 「月灯り」のための設計哲学

## 最重要原則: 判断に迷ったときの基準

1. **「このシステムは、プレイヤーがカフェの店主であるという体験を強化するか？」** — YESなら採用、NOなら再考
2. **「このシステムを取り除いても物語体験は損なわれないか？」** — 損なわれないなら本当に必要か再考
3. **「プレイヤーが1週間休んで戻ってきたとき、罰せられた気持ちになるか？」** — なるなら設計を修正
4. **「3ヶ月後もプレイヤーがキャラを好きでいられるか？」** — キャラの魅力がシステムに潰されていないか確認

---

## 1. エンゲージメントループの三層構造

すべてのシステム設計はこの三層に当てはめて検証する。

```
【ミクロループ】 1回のプレイ（1-3分）
  操作 → 結果の確認 → 小さな報酬 → 次の操作への動機
  ★ 1回の操作が「意味ある選択」を含んでいるか？

【デイリーループ】 1日の流れ（10-20分）
  ログイン → デイリーミッション → ストーリー or キャラ交流 → 明日への期待
  ★ セッション終了時に「明日やりたいこと」が明確か？

【マクロループ】 週・月単位の進行
  章クリア → 新レシピ解放 → キャラ関係深化 → 新ストーリー → 世界の変化
  ★ 週単位の目標があるか？
```

---

## 2. 深みと複雑さのバランス

### 「学びやすく、極めがいのある」三層設計

```
【表層】 誰でもわかる — メニューを選んで客に出す
【中層】 気づくと面白い — 客の好みを覚えると報酬UP、組み合わせでボーナス
【深層】 極めると楽しい — 最適なメニュー構成、隠しレシピの発見条件
```

### 戦略的選択の導入チェックリスト

- [ ] 操作の種類で消費量が変わるか（通常営業=15、特別メニュー=25、会話=10）
- [ ] メニュー構成の計画性はあるか（どのメニューを並べるかで来客傾向が変わる）
- [ ] 時間帯の概念はあるか（朝・昼・夜で客層が変わる）
- [ ] 失敗にも面白さがあるか（変な組み合わせ→キャラの面白いリアクション）

## 3. 親密度システム — 数値ではなく「関係性の質」

各ランクで関係性の質が変化する設計にする:

| ランク | 関係性 | 体験 |
|--------|--------|------|
| ★1 | 常連客 | 挨拶だけ |
| ★2 | 顔見知り | 好みが分かるようになる |
| ★3 | 友人 | 悩みを相談される、カフェ外の物語が見える |
| ★4 | 親友 | キャラの過去が明かされる、特別イベント解放 |
| ★5 | かけがえのない存在 | エンディング分岐に影響 |

### 親密度獲得手段の多様化

- 好きなメニューを出す（既存）
- 会話中の選択肢で共感を示す
- キャラ関連のサブクエストを達成する
- 他キャラとの関係を通じて理解が深まる（キャラA★3でキャラBの新しい側面が見える）

---

## 4. スタミナ設計 — 「壁」ではなく「リズム」

- スタミナは生活リズムとして設計する。Animal Crossingの「また明日ね」感覚
- スタミナ切れを「罰」ではなく「今日はここまで」にする
- **スタミナ不要の行動も用意する** — 内装変更、レシピ図鑑閲覧、キャラプロフィール確認
- スタミナ上限到達時のログインで自動ボーナス変換

---

## 5. レシピシステムの深化

- **基本レシピ**（ストーリー解放）＋ **派生レシピ**（素材の組み合わせ実験で発見）
- **季節・天候・時間帯による変化** — 雨の日限定、冬の温かいドリンク、夜限定メニュー
- **キャラ固有レシピ** — 親密度到達でキャラが「こんな飲み物があったらいいな」→ 特別レシピ解放
- **失敗の面白さ** — 変な組み合わせでも「謎の液体」として出せる。罰ではなく笑いに

---

## 6. 経済バランス

### 通貨設計の原則

```
【基本通貨: コイン】 日常の努力が報われる。獲得=売上、用途=食材・内装・レシピ
【希少通貨: 月のしずく】 特別な瞬間用。獲得=ストーリー達成・特別ミッション
【素材リソース】 コレクション欲と実験の楽しさ
```

### 報酬テーブルの心得

```
デイリー収入: プレイヤーの必要経費の120-150%（少し余裕がある）
週間目標: デイリーの2-3日分相当のボーナス
ストーリー報酬: 新機能解放 + 通貨 + 演出（大きな達成感）
```

---

## 7. 報酬の4類型

| 類型 | 例 | 役割 |
|------|-----|------|
| 確定報酬 | デイリーミッション → 固定コイン | 安心感、基盤 |
| 進捗報酬 | 「100杯達成」称号、図鑑の埋まり | 達成感、コレクション欲 |
| 発見報酬 | 隠しレシピ、天候限定イベント | 驚き、口コミ発生 |
| 感情報酬 | キャラの笑顔、感謝の言葉 | 最も強力。「遊んでよかった」と思わせる |

---

## 8. ログインボーナス設計

- **連続ではなく累計方式** — 1日休んでもリセットしない。離脱の最大原因を排除
- **7日サイクル × 4週間** — 各週にテーマ（食材→コイン→内装→ストーリー関連）
- **28日目は選択報酬** — 3つから選べる。自己決定感
- **キャラからの日替わりメッセージ** — 「今日は雨ですね。温かい飲み物が飲みたい気分です」

---

## 9. 長期エンゲージメント（エンドゲーム設計）

ストーリー完結後もプレイヤーが留まる理由:

- **後日談エピソード** — 各キャラ★5達成後のエピローグ
- **真エンディング** — 全キャラ★5 + 特定条件で最終ストーリー
- **日常モード** — メイン完結後、キャラたちの「その後の日常」が日替わりで見られる
- **シーズナルイベント** — 現実の季節・行事と連動。限定ストーリーでキャラの新しい一面

### コンテンツ更新リズム

```
毎日:    デイリーミッション更新、キャラ日替わり会話
毎週:    週間ミッション、週替わり特別客
隔週:    新レシピ or 小規模イベント
毎月:    新ストーリー or 中規模イベント
四半期:  大型イベント、新キャラの可能性
```

## 10. 絶対に避けるべきアンチパターン

### 搾取的設計

- **❌ 天井なしガチャ** — このゲームにガチャは不要
- **❌ スタミナ枯渇による課金誘導** — スタミナは「リズム」であって「壁」ではない
- **❌ FOMO（取り逃し恐怖）を煽る限定設計** — 「来年また会えます」の安心感を
- **❌ ログインボーナスの連続リセット** — 累計方式にする

### 空虚な設計

- **❌ 意味のない周回** — 同じ操作の無限繰り返し
- **❌ 水増しされた数値** — ★4→★5が指数的増加
- **❌ 見せかけの選択肢** — どれを選んでも結果が同じ
- **❌ 一度見たストーリーを読み返せない**

### バランス崩壊

- **❌ 序盤の大盤振る舞い → 後半の枯渇**
- **❌ パワーインフレ** — 新しいものが常に前より「強い」
- **❌ 救済策のない行き詰まり** — ヒントや代替ルートがない

## 物語とシステムの融合チェックリスト

新しいシステムを追加・変更する際、以下を確認する:

- [ ] カフェという空間の本質（人が集い、語らい、癒される場所）を体現しているか
- [ ] ストーリーがカフェの状況を変えるか（キャラの気分で注文が変わるなど）
- [ ] カフェの変化がストーリーに影響するか（内装変更にキャラがコメントなど）
- [ ] 「月」のモチーフを活かしているか（満月の夜だけの会話など）
- [ ] 季節の移ろいを感じさせるか
