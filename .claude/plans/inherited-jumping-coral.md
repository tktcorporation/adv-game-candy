# 学マスの「感触」を再現する — ゲームシステムUX改修

## Context

ユーザーのフィードバック:
- 「ゲームシステム部分が面白くない感じがする」
- 「ルールも法則も何をすれば良いのかもよくわからない」
- 「システムを真似して何かに寄せ切って」

### 現状分析

このゲームは既に**学マスの構造を完全に持っている**:

| 学マス | 月灯り | ファイル |
|--------|--------|---------|
| Vo/Da/Vi 3軸 | trust/understanding/empathy | `state.mbt` |
| スキルカード | 行動カード(34枚) | `cards.mbt` |
| レッスン選択 | 手札4枚から選択 | `app_action.mbt` |
| 好調/集中バフ | comfort/curiosity/courage (3種×3stack) | `logic_actions.mbt` |
| 中間/最終試験 | 週チャレンジ(7日ごと) | `event_system.mbt` |
| メモリー/サポカ | 思い出(18種、3スロット装備) | `memories.mbt` |
| 体力管理 | 心の余裕(8/日) | `state.mbt` |

**問題は「構造」ではなく「感触（feel）」。** 学マスが面白い理由は3つ:
1. **ゴールが常に見える**（「Sライン: Vo200」が画面上部に常時表示）
2. **コンボが爆発する**（「好調→集中→Perfect!」で数値が跳ねる）
3. **失敗の恐怖がある**（試験不合格 = プロデュース失敗）

月灯りにはこの3つが全て欠けている。

---

## 修正方針: ロジック変更は最小限、UX/演出で感触を変える

既存のゲームロジック（バフ計算、週チャレンジ判定、カード効果値）は**そのまま**使う。
変えるのは「見せ方」と「フィードバック」。

---

## 修正内容

### 1. 週チャレンジを「試験」として画面中央に据える

**学マスの再現**: プロデュース画面の上部に「最終試験まで○週」「Sライン: Vo200」が常時表示。

**変更箇所**: `src/ui/app_action.mbt` の週チャレンジ目標表示

**現在**:
```
第1週（残り4日）S目標: 各13 / 現在 信12 理8 共7
```
小さなテキストで1行。情報が詰め込まれて読みにくい。

**変更後** — 3軸ゲージ + Sライン表示:
```
── 第1週 試験まで4日 ──
信頼  ████████░░░░░  12/13
理解  ██████░░░░░░░   8/13
共感  █████░░░░░░░░   7/13
        ───── S ─────
```

各軸に個別のゲージを表示。Sラインを「━━ S ━━」で可視化。
「今どの軸が足りないか」が一目でわかる。

**実装**: `src/ui/app_action.mbt` L58-62 の `weekly-goal-mini` div を書き換え。
ゲージ生成関数を `app_action.mbt` 内に追加（既存の `build_akari_gauge` パターンを再利用）。

**再利用する既存関数**:
- `@game.get_week_number(day)` — `event_system.mbt:7`
- `@game.is_week_end(day)` — `event_system.mbt:12`
- state の `week_start_trust/understanding/empathy` — `state.mbt`

### 2. カードに「メインステータス」ラベルを追加

**学マスの再現**: レッスンを選ぶ ＝「今回はVoを伸ばす」という明確な意図。

**現在**: カード名 + 説明 + 「信+4 理+1 共+1」の数値。どれがメインかわからない。

**変更後** — カードヘッダーに軸アイコン:
```
⚡ 近所を散歩                     1余裕
「少し歩こう。気分転換になる。」
◆信頼 +4  理+1 共+1
付与: ⚡勇気
```

- メインステータス（最も高い値）を `◆` で強調
- カードヘッダー左に対応バフのアイコンを表示
- 「このカードは信頼メインだ」が即座にわかる

**実装**: `src/ui/app_action.mbt` の `render_action_card()` 内。
カードの3軸値を比較してメインを判定する関数を追加。

### 3. バフコンボ演出の強化（DailyScene）

**学マスの再現**: 「好調→集中→Perfect!」の演出。スコアが跳ね上がる爽快感。

**現在** (`app_scene.mbt:13-19`):
```
✦ バフ消費 → 効果1.5倍！
```
1行のテキスト。数字の変化が小さい。

**変更後** — 段階的な演出:
```
⚡ 勇気が弾けた！

信頼  8 → ×1.5 → 12！
理解  3 → ×1.5 →  4
共感  3 → ×1.5 →  4

────── COMBO ──────
♡ +20  灯P +3  コイン +20
────────────────────

「勇気を振り絞った。一歩踏み出せた気がする。」
```

- 「COMBO」という明確なラベル
- 各軸の計算過程を見せる（`8 → ×1.5 → 12！`）
- バフ未消費時は通常表示のまま（差を感じさせる）

**実装**: `src/ui/app_scene.mbt` の `render_daily_scene_view()` を書き換え。
`last_card_result` にバフ消費前の値も必要 → `CardResult` 型に `base_trust/base_understanding/base_empathy` を追加。

**変更が必要な型**: `src/data/types.mbt` の `CardResult` に3フィールド追加。
**変更が必要なロジック**: `src/game/logic_actions.mbt:96-104` で base 値も記録。

### 4. Cランクにペナルティを追加（軽い）

**学マスの再現**: 試験不合格の緊張感。

**現在**: Cランクでも灯P +2。何も困らない。
**変更後**: Cランクの場合、**翌週のスタミナが -1**（8 → 7）。

```
週チャレンジ: Cランク... +2灯P +20コイン
「疲れが溜まっている。来週は少し余裕がない。」
→ 翌週の心の余裕: 7（通常8）
```

- Sランクボーナス: 翌週スタミナ +1（8 → 9）で報酬感も追加
- A/Bは変更なし（8のまま）
- ゲーム序盤（Week 1-2）は免除（チュートリアル期間）

**実装**:
- `src/game/event_system.mbt` に `stamina_modifier` を追加
- `src/game/logic_daily.mbt` の `advance_to_next_day()` でスタミナ回復時に modifier 適用
- `src/game/state.mbt` に `next_week_stamina_mod: @signals.Signal[Int]` 追加

### 5. Home画面の「今週の目標」を試験風に

**現在** (`app_home.mbt:33-46`):
```
── 今週の目標 ──
第1週（残り4日）S目標: 各13 / 現在 信12 理8 共7
```

**変更後**:
```
── 第1週 試験まで4日 ──
S目標: 信13 理13 共13
現在:  信12 理 8 共 7
       ▲OK  △-5  △-6
```

- 不足分を `△-5` で明示
- 達成済みの軸は `▲OK` で安心感

**実装**: `src/ui/app_home.mbt` の weekly-event-goal div を書き換え。
表示生成ロジックは `event_system.mbt` に `get_weekly_goal_detail()` として追加。
