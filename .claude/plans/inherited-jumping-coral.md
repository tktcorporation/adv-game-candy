# ブルアカ風ロードマップ再設計 — 通知バッジ + ロードマップパネル削除

## Context

ユーザーから2つのフィードバック:
1. 「ゲームのゴールと物語のゴールを一致させすぎないで」
2. 「いっぱいあっても困る。ブルアカみたいな設計にして」

ブルアカの設計思想: ホーム画面にストーリー進捗ダッシュボードは**ない**。新コンテンツは**通知バッジ（NEW!や赤●）**で知らせる。日常のゲームプレイ自体がゴールで、ストーリーは遊んだご褒美として自然に解放される。

調査の結果、このゲームは**既にブルアカ的な構造を持っている**:
- メッセージログで新章解放を通知（`logic.mbt:67-70`）
- DayResultで「灯で新しい物語が読めます！」と案内（`app_day_result.mbt:81`）
- VisitSceneで条件付き「ストーリーを読む」ボタン（`app_scene.mbt:165-189`）

**問題は、前回追加したロードマップパネル（路線図 `●━●━◎━○━○`）がこの設計と矛盾していること。**

---

## 修正内容

### 1. ロードマップパネルを削除

前回追加した `render_roadmap(state)` の呼び出しをHome画面から削除。`app_roadmap.mbt` ファイル自体も削除。CSSの `.roadmap-*` クラスも削除。

**理由**: ブルアカのホーム画面に進捗ダッシュボードは存在しない。

### 2. 「灯に行く」ボタンに NEW! バッジ

新しいストーリーが解放されているとき、灯ボタンのテキストに `✦` マークを追加。

```
通常:     [ 灯に行く（3/10灯P）]
解放時:   [ ✦ 灯に行く（12/10灯P）]
```

**判定条件**: 次章の `unlock_visit_count` ≤ `total_visits` かつ灯P十分。
**実装**: `app_home.mbt` L106-112 のボタンテキスト `text_dyn` 内で条件分岐。

### 3. アルバムボタンに NEW バッジ

新しいストーリーが解放されている、または最近思い出を獲得したとき、「アルバム」ボタンのテキストに `●` を付与。

```
通常:     [ アルバム ]
新情報時: [ アルバム ● ]
```

**判定条件**: 次章解放済み（ストーリータブに新情報）
**実装**: `app_home.mbt` L199-203 のボタンテキストで条件分岐。

### 4. 灯Pゲージをボタン直下にインライン表示（小さく）

ロードマップパネル内の灯Pゲージ `▓▓▓░░░░░░░` は、プレイヤーが「灯に行けるまでどれくらいか」を直感的に把握できて良かった。これをロードマップパネルからは外すが、「灯に行く」ボタンのすぐ下に**1行の小さなゲージ**として残す。

```
[ 灯に行く（3/10灯P）]
  ▓▓▓░░░░░░░
```

**実装**: `app_home.mbt` の灯ボタンの直後に `@dom.div` を追加。
